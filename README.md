# TopIAM Universal Form Interceptor

> 一款面向 TopIAM 场景的浏览器扩展：自动接管表单代填跳转、跨系统登录链路与会话状态。

[![Platform](https://img.shields.io/badge/Platform-Chromium%20Extensions-2ea44f)](#安装与使用)

## 项目简介

本项目完全独立于TopIAM项目，不用更改你的SSO平台代码，用于增强 TopIAM 发起的表单代填应用登录体验：

- 自动配置绑定sso平台域名，发现应用列表与登录链路
- 拦截表单提交
- 在目标系统页面执行智能表单填充
- 维护 SSO 会话状态，并在失效时触发保护策略

适用于企业内部 SSO 场景下“多系统、多登录页、不统一表单结构”的复杂环境。

---

## 功能特性

- 自动发现应用
	- 在 TopIAM 页面自动识别应用条目与启动参数，更新已发现应用列表
- 登录链路接管
	- 支持 `initiator` 预取、POST 请求体提取、目标页任务派发
- 智能代填
	- 针对多种用户名/密码字段命名进行识别与填写
	- 支持多 frame 场景与页面重试恢复
- 会话治理
	- 同步 TopIAM 用户身份与认证状态
	- 登录态心跳检测、失效后全局会话清理
- 安全增强
	- 凭据短期内存保管（TTL）
	- 本地缓存凭据采用 AES-GCM 加密

---

## 工作原理（简版）

1. 用户在 TopIAM 门户点击应用
2. 扩展识别启动请求并尝试预取 `initLoginUrl`
3. 提取用户名/密码/附加参数后，签发临时任务
4. 跳转到目标系统并在目标页面完成自动填充
5. 后台持续监控 TopIAM 会话状态，必要时清理会话并阻断访问

## 安装与使用

### 1) 本地加载（Chrome / Edge）

1. 打开浏览器扩展管理页（如 `chrome://extensions`）
2. 开启“开发者模式”
3. 点击“加载已解压的扩展程序”
4. **选择 `extension/` 目录**（不是仓库根目录）

### 2) 基础配置

点击扩展图标，在弹窗中配置：

- TopIAM 域名列表（支持逗号分隔多个域名, 插件会自动发现你的sso平台）
- 是否启用调试面板（仅 `/portal/app`）

### 3) 日常使用

- 在 TopIAM 页面正常点击应用入口
- 扩展会自动接管复杂登录链路并完成代填

## 权限说明（Manifest）

当前扩展使用以下关键权限：

- `storage`：保存域名、已发现应用、加密缓存等配置
- `tabs` / `scripting`：创建与控制标签页，注入脚本完成提取与填充
- `webNavigation`：监听导航链路，识别登录接管时机
- `webRequest`：捕获关键请求（如表单 POST、退出信号）
- `cookies`：读取/清理指定站点 cookie 以维护会话状态
- `activeTab`：与当前活动标签页交互
- `alarms`：定时清理与状态巡检

`host_permissions` 当前为 `http://*/*` 与 `https://*/*`，用于兼容多应用、多域名场景。

## 安全与隐私

- 凭据主要通过短期任务机制在内存中传递，超时后自动失效
- 本地凭据缓存使用 AES-GCM 加密（用于预取失败回退等场景）
- 建议仅在受信任的企业终端与网络环境中使用
- 建议在生产部署前进行内部安全审计与最小权限评估

> 注意：本项目涉及登录凭据处理，请务必遵循组织的安全合规要求。

## 常见问题（FAQ）

### Q1：点击应用后没有自动代填

- 确认 TopIAM 域名配置正确
- 确认已从 `extension/` 目录加载扩展
- 检查目标页面是否存在特殊 CSP 或登录控件变体

### Q2：提示会话失效或被拦截

- 先确认 TopIAM 本身是否仍处于登录状态
- 检查是否触发了全局会话清理（如退出登录、心跳失效）

### Q3：应用识别列表为空

- 先在 TopIAM 门户点击一次应用，等待自动发现
- 必要时在弹窗中清空缓存后重新触发
